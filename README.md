# Monitoring Stack Documentation

## Overview

This monitoring stack provides complete observability for infrastructure, containers, databases, Redis, and application endpoints using Prometheus, Grafana, Alertmanager, and multiple exporters.

### Components

| Component         | Purpose                              |
| ----------------- | ------------------------------------ |
| Prometheus        | Metrics collection and storage       |
| Grafana           | Visualization and dashboards         |
| Alertmanager      | Alert routing and notifications      |
| Node Exporter     | Linux server monitoring              |
| cAdvisor          | Docker container monitoring          |
| Redis Exporter    | Redis monitoring                     |
| DB Exporter       | Database monitoring                  |
| Blackbox Exporter | Endpoint and availability monitoring |

---

# Components Description

## 1. Prometheus

### Purpose

Prometheus is responsible for collecting, storing, and querying metrics from all exporters.

### Responsibilities

* Scrape metrics from exporters
* Store time-series data
* Execute PromQL queries
* Trigger alerts

### Example Metrics

```promql
up
```

```promql
node_cpu_seconds_total
```

```promql
container_memory_usage_bytes
```

---

## 2. Grafana

### Purpose

Grafana provides dashboards and visualizations for all collected metrics.

### Dashboard Categories

* CPU Usage

#### Container Dashboard

#### Database Dashboard

* Active Connections

* Connected Clients

## 3. Alertmanager

### Purpose

Alertmanager handles alerts generated by Prometheus.

### Supported Notifications

* Slack
* Microsoft Teams
* Telegram
* PagerDuty


#### High Memory

```yaml

```

#### Disk Utilization

```yaml

```

## Alert Coverage

The monitoring stack includes proactive alerting across infrastructure, containers, databases, Redis, endpoints, and BuildPiper services.

### Infrastructure Alerts

* Node Exporter availability monitoring
* High CPU utilization (>80%)
* High memory utilization (>80%)
* Disk space warning (>75%)
* Disk space critical (>80%)
* Network transmit/receive errors

### Database Alerts

* MySQL service down
* MySQL TCP connectivity failure
* High database connections (Warning/Critical)

### Redis Alerts

* Redis service down
* Redis TCP connectivity failure
* Low connected clients
* Redis reconnect storms
* High memory utilization
* Redis key evictions
* Low command processing rate

### Endpoint & SSL Alerts

* Application endpoint down
* Slow API response times
* SSL certificate expiry (<30 days)
* Critical SSL expiry (<7 days)

### Container Alerts

* BuildPiper container down
* Container restarts detected
* High container CPU usage
* High container memory usage

### Celery & Queue Alerts

* High queue depth
* Critical queue backlog
* Task failure spikes
* Excessive retry tasks
* No workers online
* High worker concurrency

### Alert Severity Levels

| Severity | Purpose                                             |
| -------- | --------------------------------------------------- |
| Warning  | Early indication of potential issues                |
| Critical | Immediate action required to prevent service impact |

### Notification Channels

Alerts are generated by Prometheus and routed through Alertmanager to configured notification channels such as:

* Email
* Slack
* Microsoft Teams
* Webhooks
